# Specs

Antes de implementar qualquer feature não trivial, crie um arquivo `.md` nesta pasta com a especificação.

## Formato

```md
# Spec: <nome da feature>

> Uma linha descrevendo o objetivo.

## Contexto

Por que essa feature existe. O que o usuário precisa.

## Pesquisa / Opções

Liste as alternativas consideradas com prós, contras e conclusão.
Inclua referências externas se relevante.

## Decisão

Qual opção foi escolhida e por quê.

## Especificação de Implementação

Arquivos a criar/modificar, interfaces, pseudo-código relevante.

## To-dos de Implementação

- [ ] Tarefas concretas para executar a spec
```

## Regras

- Um arquivo por feature — nome em kebab-case (`feature-name.md`)
- Pesquise antes de decidir — documente o que foi descartado e o motivo
- Mantenha a spec atualizada se a implementação mudar
- Não comece a implementar sem a spec aprovada
